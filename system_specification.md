# 市区町村訪問マスター V9.5 システム仕様書

## 1. システム概要
本システムは、ユーザーが国内の市区町村を訪れた記録を入力・管理し、その達成率を視覚的に確認できるWebアプリケーションです。4年間を「1周期」とする独自の期間管理アルゴリズムを採用しており、基準日からの経過年数（1年目〜4年目、または過去）に応じて訪問記録を色分けして表示します。設定や記録データはブラウザローカルストレージ（Local Storage）に保存されるとともに、Firebase (Firestore) を通じてクラウドにリアルタイム同期されるため、異なるブラウザ間でも同じデータを管理できます。

## 2. 開発・実行環境
*   **フロントエンド**: HTML5, CSS3, JavaScript (Vanilla JS)
*   **CSSフレームワーク**: Tailwind CSS (CDN経由)
*   **フォント**: Noto Sans JP
*   **データ保存機能**: Firebase Cloud Firestore (クロスブラウザ同期) および Web Storage API (localStorage バックアップ)

## 3. データ構造
アプリケーションは以下のデータを保持・管理します。

### 3.1 訪問記録データ (`visit_v95_recs`)
訪問地の記録を格納する配列です。各要素は以下のプロパティを持ちます。
*   `id` (Number): レコードの一意な識別子（作成時のタイムスタンプ）
*   `pref` (String): 都道府県名
*   `city` (String): 市区町村名
*   `date` (String): 訪問日（YYYY-MM-DD形式）

### 3.2 設定データ (`visit_v95_cfg`)
システムの動作設定データを保持するオブジェクトです。
*   `baseStart` (String): 周期の基準開始日（デフォルト: '2022-04-01'）
*   `prefDenominators` (Object): 各都道府県をキーとした、分母（全市区町村数）のハッシュマップ

## 4. 主要機能

### 4.1 記録入力機能 (Record Entry)
*   **都道府県の選択**: Datalistを用いたサジェスト入力。
*   **市区町村の入力**: テキスト直打ちによる入力。
*   **重複チェック機能**: 市区町村名を入力時、既に同一都道府県内に同名の訪問記録が存在する場合、「⚠️ 過去訪問アリ：[前回訪問日]」という警告をリアルタイムで表示します。
*   **訪問日の入力**: カレンダー（date型インプット）による日付選択。

### 4.2 周期管理機能 (Cycle Management)
*   **周期の定義**: 基準開始日（`baseStart`）から4年間を「1周期」と定義します（例：第1周期、第2周期...）。
*   **進行中周期の自動判定**: 現在の日付と基準開始日の差分から、アクティブな周期（最大第4周期まで）を自動算出します。
*   **訪問スケジュールの判定**: 選択されている該当周期の開始日を起点に、訪問記録の年月を計算して以下の区分に分類し、UIデザイン（バッジ色・枠線）を変化させます。
    *   **1年目 (1ST YR)**: 経過月数 0〜11ヶ月 (ローズ色)
    *   **2年目 (2ND YR)**: 経過月数 12〜23ヶ月 (イエロー)
    *   **3年目 (3RD YR)**: 経過月数 24〜35ヶ月 (グリーン)
    *   **4年目 (4TH YR)**: 経過月数 36〜47ヶ月 (水色)
    *   **過去・対象外 (PAST)**: 経過月数がマイナス、または48ヶ月以上 (グレー)
*   **周期の切り替え**: ヘッダーのタブボタンより、第1周期〜第4周期の表示を任意に切り替えることが可能です。

### 4.3 ダッシュボード・進捗・統計機能 (Progress & Stats)
*   **全体の達成率**: 記録されている全データの「重複を排除した訪問数」と「全国総分母」を元に、全体の達成状況を％表示、およびプログレスバーで視覚化します。
*   **全期間 実質訪問数**: （ユニーク市区町村数 / 総分母数）を分数値として表示します。

### 4.4 都道府県別ビュー (Prefecture Details)
*   **都道府県リスト**: 左サイドバーに47都道府県別の各進捗率（％）とプログレスバーを一覧表示します。
*   **訪問地グリッド表示**: サイドバーで選択した都道府県内の「アクセス済み市区町村」をグリッド形式（タイル状）で表示します。
    *   同一市町村に複数回訪れた場合は、最新の訪問記録が優先されて周期判定の色が適用されます。
*   **分母分子表示**: 選択中の都道府県の「到達済み数 / その都道府県の分母」を表示します。

### 4.5 訪問ログ表示 (Visit Log)
*   全訪問履歴をテーブル形式にて「古い順（昇順）」で一覧表示します（連番、日付、区分バッジ、場所を表示）。
*   ログにある各レコードに設けられた「✕」ボタンから、個別に記録を削除することができます（削除時はブラウザの確認ダイアログが表示されます）。

### 4.6 システム設定機能 (System Settings)
ヘッダーの「⚙️ 周期・分母設定」ボタンからモーダルを開き、以下の各項目を設定できます。
*   **周期の基準開始日**: 全計算の基本となる日付を変更可能。
*   **都道府県ごとの分母変更**: 市区町村の統廃合などを想定し、各都道府県内の総市区町村数を個別で変更可能。
*   **全市区町村 総分母**: 各都道府県の分母の入力合計値がリアルタイムで表示されます。

## 5. UI/UXの特色
*   **ダイナミックで活気のあるデザイン**: Tailwind CSSを利用し、角丸(rounded-2xl)やドロップシャドウを多用したモダンなカードデザイン。
*   **カラフルな状態管理**: 周期ごとの鮮やかで明瞭なカラーリング（赤、黄、緑、青）による視覚的な直感性。
*   **スムーズな操作**: SPA（Single Page Application）として構築されており、入力・削除・設定変更などが画面遷移（リロード）を伴わずにリアルタイムで描画内容へ反映されます。
*   **マルチデバイス同期**: Firebaseのリアルタイムリスナー機能により、他のデバイス上でデータが更新された場合も、手元の画面に自動的にデータが反映されます。
